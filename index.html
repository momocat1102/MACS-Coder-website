MACS-Coder

A Multi-Agent Coding Framework for Small LMs — From Fast Thinking to Deep Planning

Introduction

Large Language Models (LLMs) have made significant strides in code generation, yet solving complex programming tasks remains challenging. Current state-of-the-art multi-agent frameworks often use resource-intensive, one-size-fits-all strategies. We introduce MACS-Coder (Multi-Agent Adaptive Coding Structure), a dual-process framework inspired by human cognition that dynamically adapts its problem-solving strategy. It uses a fast subsystem for simpler tasks and a deep subsystem with specialized planning, coding, and debugging agents for complex challenges.

MACS-Coder achieves new SOTA results with remarkable efficiency. Using gpt-oss-20B, it attains 99.4% on HumanEval, 93.2% on MBPP, and 83.2% on LiveCodeBench V5, consistently outperforming CodeSIM and MapCoder in both accuracy and computational efficiency. The primary contribution is bridging the performance gap between compact open-source models and elite closed-source systems—demonstrating that an open-source gpt-oss-20B model with MACS-Coder can achieve performance comparable to top-tier models like GPT-4 and Gemini-2.5 Pro.

MACS-Coder Overview

MACS-Coder introduces an efficient dual-stage "Fast and Deep Thinking" framework inspired by human cognitive systems. The Fast-Thinking stage quickly handles simple problems with minimal resources, while the Deep-Thinking stage provides comprehensive reasoning for complex challenges through specialized agents: Planning, Coding, Debugging, and STD IO handling.

Fast Thinking System

Designed for simple problems, this stage generates solutions directly without extensive computation. It first attempts direct code generation and only transitions to Deep Thinking if initial tests fail, conserving resources for truly challenging problems.

Deep Thinking System

For complex problems, this stage employs four specialized agents working in sequence:

  • STD IO Tool: Creates structured input/output templates for competitive programming problems, improving code readability and correctness.
  • Planning Agent: Generates comprehensive algorithmic plans, evaluates multiple solutions, and selects the most stable approach.
  • Coding Agent: Implements the plan with detailed comments, ensuring adherence to the strategy outlined in the planning phase.
  • Debugging Agent: Handles three types of errors through specialized blocks: STD IO errors (input/output formatting), Runtime errors (execution failures), and Wrong Answer errors (logic corrections).


Results

double_arrowCompetitive Problem Solving

Performance comparison on the HumanEval and MBPP benchmarks using open-source models (gpt-oss-20B, Qwen3-14B, and Qwen3-8B). The evaluation metrics include: ACC (Pass@1 Accuracy), Time (average seconds per problem), Tokens (average generated response tokens per problem), and ET ACC (a more stringent evaluation with additional private test cases). The results show that MACS-Coder not only achieves the highest accuracy but also demonstrates superior efficiency by consuming fewer Tokens than other high-performing methods like CodeSIM and MapCoder.

MACS-Coder redefines competitive-level code generation with both accuracy and efficiency. Powered by its dual “Fast and Deep Thinking” framework, it achieves 83.2% accuracy on gpt-oss-20B—setting a new SOTA and surpassing CodeSIM (80.4%) and MapCoder (77.8%). At the same time, it saves 18% in token usage on hard problems, showing that higher accuracy doesn’t need higher cost. Remarkably, MACS-Coder also empowers smaller models: Qwen3-8B with MACS-Coder reaches 63.1%, even outperforming the larger Qwen3-14B with MapCoder (62.2%). These results highlight MACS-Coder’s ability to make problem-solving not only smarter, but also more resource-efficient.

double_arrowBasic Code Generation

Performance comparison on the HumanEval and MBPP benchmarks using open-source models (gpt-oss-20B, Qwen3-14B, and Qwen3-8B). The evaluation metrics include: ACC (Pass@1 Accuracy), Time (average seconds per problem), Tokens (average generated response tokens per problem), and ET ACC (a more stringent evaluation with additional private test cases). The results show that MACS-Coder not only achieves the highest accuracy but also demonstrates superior efficiency by consuming fewer Tokens than other high-performing methods like CodeSIM and MapCoder.

In basic code generation tasks, MACS-Coder excels by delivering SOTA-level accuracy with fewer resources. On HumanEval, Qwen3-8B with MACS-Coder achieves 89.0% accuracy while cutting both time (26.8s) and token usage (1,711) compared to CodeSIM. Even more impressively, this efficiency lets smaller models punch above their weight: Qwen3-8B + MACS-Coder matches the performance of the larger Qwen3-14B using CoT (89.0%). This demonstrates how MACS-Coder brings top-tier results with practical, resource-friendly models—making advanced code generation more accessible and cost-effective.

double_arrowComparison with State-of-the-Art Models

Effectiveness of MACS-Coder on LiveCodeBench V5. Compared to their base versions, models enhanced with our method show substantial improvements in Pass@1 scores: gpt-oss-120b (+6.0), gpt-oss-20b (+15.9), Qwen3 14B (+25.1), and Qwen3 8B (+22.7). This demonstrates the significant performance gains provided by MACS-Coder.

On the LiveCodeBench benchmark, MACS-Coder proves its absolute competitiveness by elevating open-source models to match top proprietary systems. With gpt-oss-20B, MACS-Coder achieves 83.2% Pass@1, surpassing Grok-3-Mini (81.4%) and O3-Mini (80.5%), and coming close to Gemini-2.5 Pro (84.7%). Crucially, MACS-Coder also empowers smaller models: Qwen3-14B + MACS-Coder (68.6%) outperforms the unenhanced gpt-oss-20B (67.3%) and rivals O1-Mini (68.4%). Even Qwen3-8B + MACS-Coder (63.1%) beats much larger models like DeepSeek V3 (56.3%) and Claude-3.5-Sonnet (51.5%). These results highlight MACS-Coder as a powerful enhancer that makes SOTA-level performance more accessible, enabling open-source models to tackle challenges once reserved for top proprietary systems.

double_arrowPerformance-Cost Analysis

Performance comparison between MACS-Coder (solid lines) and CodeSIM (dashed lines) across multiple planning iterations. MACS-Coder maintains a substantial performance advantage over CodeSIM regardless of the iteration count, demonstrating its superior effectiveness. While both methods benefit from more iterations, MACS-Coder consistently operates at a higher performance level.

To highlight its efficiency, we compared MACS-Coder with CodeSIM on accuracy vs. token usage. Across all benchmarks, the results are clear: MACS-Coder delivers higher accuracy with fewer tokens. On LiveCodeBench, Qwen3-8B + MACS-Coder improves accuracy by +8.7% while saving 10k+ tokens, and even gpt-oss-20B sees gains of +2.8% accuracy with 4,400 fewer tokens. The improvement consistently moves performance toward the top-left (more accurate, more efficient) in every case. This advantage comes from MACS-Coder’s Fast and Deep Thinking framework, which avoids wasting computation on simple problems and focuses resources on harder ones—making it not just more accurate, but also far more efficient for real-world use.
